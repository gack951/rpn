# エミュレータで確認したRealCalcの挙動
RealCalcの挙動は本プロジェクトの基準となるので、エージェントが確認したことは以下に追記する。

## ボタン操作の視覚的フィードバック
ボタンを押している間はボタン背景色が黄緑色になる。長押しでSHIFT機能が発動した場合はボタン上の黄色いSHIFT機能ラベルの背景が短時間黄緑色になる。

## 例: `1/x` キーをX=0の状態で押したときの挙動

- 確認対象: `1/x` キーの役割と、Xスタックが0のときの画面表示
- 操作手順:
  1. 初期状態でXスタックが `0.` になっていることを確認する
  2. `1/x` キーを押す
- RealCalcの結果: `1/x` は逆数を求めるキーであり、X=0 の状態で押すとゼロ割となって表示部が `Error.` になる
- 補足: 今回の確認では、`1/x` 押下前の表示は `0.`、押下後の表示は `Error.` だった
- スクリーンショット保存先: `./screenshots/emulator-before-tap.png`, `./screenshots/emulator-after-tap.png`

## 追加仕様メモ

- `hyp` / `hyp^-1`
  - 公式ヘルプでは単発計算ではなく、双曲線関数モードと逆双曲線関数モードのトグルとして説明されている
  - `hyp` の後に `sin/cos/tan` で `sinh/cosh/tanh`、`hyp^-1` の後に `sin/cos/tan` で `asinh/acosh/atanh`

- `%`
  - 公式ヘルプでは RPN 時は `Y` の `X` パーセントを求め、結果を `X` に置くが `Y` は残る

- `n!`, `nCr`, `nPr`
  - それぞれ階乗、組合せ、順列
  - `nCr` と `nPr` は 2 項演算として `Y` と `X` を使う

- `FSE`
  - 公式ヘルプでは表示形式を `Normal -> Fix -> Sci -> Eng` の順で切り替える

- `TAB`
  - RPN の公式ヘルプではスタック内容をダイアログ表示するキーとして説明されている

- `RCL` / `STO`
  - 公式ヘルプでは追加メモリの呼び出し / 保存ダイアログを開く
  - `MR/MS/M+/M-` はメインメモリ、`RCL/STO` は追加メモリ

- `CNST`
  - ルートダイアログはカテゴリ一覧
  - 確認できたカテゴリ: `Universal`, `Electromagnetic`, `Atomic & Nuclear`, `Physico-Chemical`, `Other`
  - カテゴリ選択後に各定数の一覧が開き、項目を選ぶと X にその値を入力する
  - 実装では APK 内 `res/7j.xml` の定数テーブルを元に、全カテゴリ・全項目を再現する

- `CONV`
  - ルートダイアログは変換カテゴリ一覧
  - 確認できたカテゴリ: `Length`, `Area`, `Volume`, `Mass`, `Speed`, `Time`, `Force`, `Pressure`, `Energy`, `Power`, `Fuel Consumption`, `Temperature`, `Data Size`
  - カテゴリ選択後に `From Unit`、続いて `To Unit` の順でダイアログが開く
  - 変換式は APK 内 `res/7j.xml` の説明どおり `result = (input - src.offset) * (src.ratio / dst.ratio) + dst.offset`
  - `inverse="true"` の単位は、変換前または変換後に逆数処理が入る
  - 実装では APK 内 `res/7j.xml` の変換テーブルを元に、全カテゴリ・全単位を再現する

- `DEC` / `HEX` / `OCT` / `BIN`
  - 公式ヘルプでは表示と入力を基数モードへ切り替える
  - DEC に戻るまでは整数モードとして扱う

- `STACK`
  - `DROP` キーの SHIFT 側機能として、スタックダイアログを開く
